Piano di Pubblicazione dell'App
 Fase 1: Protezione con Password Globale
 Aggiungere una schermata di inserimento password su React.
 Proteggere le API del backend (Python) in modo che accettino solo la password corretta.
 Fase 2: Salvataggio su GitHub
 Creare un repository su GitHub.
 Caricare tutto il codice attuale al sicuro.
 Fase 3: Database Cloud (Supabase)
 Creare un database online gratuito (es. Supabase).
 Adattare la configurazione di Python per usare il nuovo database al posto del file 
portfolio.db
.
 Fase 4: Pubblicazione (Deploy)
 Mettere online il backend (es. su Render.com).
 Mettere online il frontend (su Vercel) e farlo comunicare con il backend.

 IDEA FUTURA: Evoluzione in SaaS Multi-Utente
 1. Sistema di Autenticazione: Implementare Supabase Auth per Login/Registrazione (Email e Password).
 2. Database Multi-Tenant: Aggiungere la colonna `user_id` in tutte le tabelle e attivare la Row Level Security (RLS) su Supabase.
 3. Backend: Passaggio del token JWT nelle chiamate FastAPI per permettere ad ogni utente l'accesso esclusivo al proprio portafoglio.
 4. Frontend: Aggiornare la gestione dello stato React al volo svuotando i dati al logout.
